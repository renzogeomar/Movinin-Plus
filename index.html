01 — Producto seleccionado

¿Qué es Movinin?

Plataforma open source de gestión de alquileres de propiedades con licencia MIT, construida con stack MERN moderno y Docker.

Movinin
↗ github.com/aelassas/movinin

Sistema web completo para gestionar el alquiler de propiedades inmobiliarias. Incluye tres frontends distintos (agencia, inquilino y administrador), una API REST con Node.js + Express, base de datos MongoDB y soporte Docker Compose. Más de 40 000 líneas de código y actividad reciente en el repositorio.

02 — Arquitectura de Sistema

Arquitectura y Componentes

Basado en la documentación técnica, Movinin sigue una arquitectura Cliente-Servidor donde múltiples frontends independientes se comunican de forma asíncrona con una API REST central y una base de datos NoSQL.

⚙️
Backend API (Express)
Servidor central en Node.js que maneja toda la lógica del negocio. Utiliza jose para JWT, bcrypt para encriptación de claves y middleware de seguridad contra ataques XSS/CSRF.
🏢
Frontend Web (React)
Aplicación web para el cliente final y agencias. Permite realizar búsquedas y reservas, interactuando directamente con las pasarelas de pago y la API.
🛡️
Admin Panel
Interfaz aislada en React para la administración global. Se conecta únicamente con el Backend API para monitorear el sistema y gestionar agencias.
📱
Mobile App
Aplicación nativa (iOS/Android) con React Native. Gestiona flujos de pago móvil y notificaciones push consumiendo los servicios externos requeridos.
💾
Persistencia y Assets
Uso de MongoDB para el almacenamiento estructurado de datos, y servicios CDN para el delivery optimizado de imágenes de vehículos y propiedades estáticas.
🔌
Servicios Externos
Integraciones críticas delegadas a terceros: Stripe y PayPal para transacciones, Expo para push, Nodemailer para correos transaccionales y Sentry para monitoreo de errores.

06 — Herramientas DevOps

Stack del proyecto

Las cuatro herramientas GitHub que integran el proceso Scrum + DevOps.

📋
GitHub Projects
Tablero Kanban/Scrum del equipo. Organiza el trabajo: qué hacer, quién lo hace y en qué estado está.
Gestionar el Product Backlog, armar sprints y ver el progreso en tiempo real.
⚡
GitHub Actions
Automatización CI/CD. Ejecuta tareas automáticas cada vez que alguien sube código al repositorio.
Compilar, correr pruebas, detectar errores y desplegar a staging automáticamente.
🌐
GitHub Pages
Servidor de publicación gratuito. Convierte el repositorio en un sitio web accesible públicamente.
Publicar la demo del sprint, documentación técnica y burndown chart.
🎫
GitHub Issues
Sistema de tickets. Cada issue representa una tarea, historia de usuario, bug o mejora.
Registrar historias de usuario del backlog y reportar bugs del sprint.
